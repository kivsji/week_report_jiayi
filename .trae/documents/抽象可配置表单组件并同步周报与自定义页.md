## 目标
- 仅在“自定义报表”新增两个可配置模块："反馈内容主要事项" 与 "本周工作与提升方案"，支持自定义标题与提示文案。
- 标签（如“总计/已完成/待解决”）可自定义标题与颜色，并可扩展状态数量。
- 抽象为统一的可配置组件，仅在自定义页复用；周报页保持现状，不做改动。

## 范围与约束
- 不修改 `components/Dashboard.tsx` 与服务层逻辑；周报页面完全保持现有实现。
- 仅修改/新增与自定义页面相关的类型、组件与页面集成。

## 数据模型（扩展 `types.ts`）
- `SectionTag = { key: string; label: string; color: string }`
- `SectionItem = { id: string; content: string; status?: string; meta?: Record<string, string> }`
- `SectionConfig = { id: string; title: string; hint?: string; tags: SectionTag[]; items: SectionItem[] }`
- 在自定义配置中新增字段：`sections: SectionConfig[]`（持久化到 `localStorage` 与 JSON 导入/导出）。

## 组件设计（仅自定义页使用）
- `components/ConfigurableSection.tsx`
  - Props：`config: SectionConfig`
  - 渲染：标题、提示文案、标签胶囊（按 `tags` 渲染并统计 `items` 中该状态数量）、内容列表（按状态分组或扁平展示，保持周报风格）。
  - 提供中文函数级注释。
- `components/SectionEditor.tsx`
  - 在自定义页弹窗使用：编辑 `title/hint`；增改删 `tags`（label/color/key）；维护 `items`（content/status/meta）及排序。
  - 校验：颜色合法（HEX）、tag key 唯一；中文函数级注释。

## 自定义报表集成（`components/CustomDashboard.tsx`）
- 增加“模块管理区”：
  - 列表展示 `sections`，卡片包含编辑、删除、上下移动排序。
  - “新增模块”打开 `SectionEditor`；保存写入到 `config.sections`，并持久化。
- 导入/导出 JSON 包含 `sections` 字段；首次无该字段时默认空数组以兼容旧配置。

## 标签颜色与样式
- 默认色板（如 `#4F46E5`、`#10B981`、`#F59E0B`、`#EF4444`）；允许用户自定义任意 HEX，非法时回退到默认色。
- 胶囊标签采用 Tailwind 风格，文本可读性优先（深色文字，浅色背景）。
