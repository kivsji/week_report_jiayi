## 目标
- 仅在“自定义报表”增加两个模块：“反馈内容主要事项”“本周工作与提升方案”。
- 模块支持自定义：标题、提示文案、状态标签（标题与颜色），内容项。
- 通过统一可配置组件实现，周报页不改动。

## 范围与约束
- 不修改 `components/Dashboard.tsx` 与服务层逻辑。
- 仅改动自定义页及其相关类型与组件，保持现有风格。

## 数据模型（扩展）
- `SectionTag = { key: string; label: string; color: string }`
- `SectionItem = { id: string; content: string; status?: string; meta?: Record<string, string> }`
- `SectionConfig = { id: string; title: string; hint?: string; tags: SectionTag[]; items: SectionItem[] }`
- 在自定义配置中新增：`sections: SectionConfig[]`，参与本地持久化与 JSON 导入/导出。

## 组件设计（仅自定义页使用）
- `ConfigurableSection.tsx`
  - props：`config: SectionConfig`
  - 渲染：标题、提示文案、标签胶囊（按 `tags` 与 `items` 统计计数）、内容列表（支持按状态分组或扁平）。
  - 提供中文函数级注释。
- `SectionEditor.tsx`
  - 弹窗编辑：`title/hint`、标签（增改删 `label/color/key`）、内容项（`content/status/meta`）、排序。
  - 校验：颜色 HEX 合法性、tag key 唯一；中文函数级注释。

## 自定义报表集成（`CustomDashboard.tsx`）
- 新增“模块管理区”：
  - 列表显示 `sections`，提供编辑、删除、上下移动排序。
  - “新增模块”打开编辑器，保存后写入 `sections` 并持久化。
- 导入/导出 JSON 时包含 `sections`；不存在该字段则默认空数组兼容旧配置。

## 样式与持久化
- 标签默认色板（如 `#4F46E5`、`#10B981`、`#F59E0B`、`#EF4444`），用户可自定义 HEX；非法时回退默认色。
- 继续使用 Tailwind 胶囊样式与卡片布局；使用现有 `useLocalStorage` 持久化。

## 验证
- 在自定义页：新增两个模块，配置标题/提示/标签与颜色，录入内容项与状态；排序、删除、导入/导出后能正确还原；本地持久化正常。
- 周报页：无改动，无影响。

- 修改/新增文件：`types.ts`（新增 Section* 类型）、`components/ConfigurableSection.tsx`、`components/SectionEditor.tsx`、`components/CustomDashboard.tsx`（集成模块管理区）。
- 不引入新依赖；所有新增函数与关键组件添加中文函数级注释。
